# PLANS.md

## Quando planejar

Use um plano antes de implementar:

- sistema de ranking;
- sistema de badges;
- regras de pontuacao;
- busca;
- refactor estrutural;
- mudancas em privacidade e visibilidade publica.
- evolucao do painel do professor para backend real.

## Formato do plano

Todo plano deve conter:

1. objetivo;
2. contexto;
3. arquivos impactados;
4. modelo de dados envolvido;
5. riscos;
6. estrategia de implementacao;
7. estrategia de teste;
8. criterios de aceite;
9. rollback.

## Exigencia adicional

Antes de alterar uma feature complexa, mapear dependencias, listar arquivos a criar ou editar, explicar impactos em dados/UI e definir como validar em build estatico.
